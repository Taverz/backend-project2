# Follow + Timeline — бизнес-требования

> Пишет человек. 3–10 строк. Никакой архитектуры.

---

## Требования

Пользователи могут подписываться друг на друга.
Лента каждого пользователя состоит из твитов тех, на кого он подписан.
Лента обновляется при создании твита (fan-out on write).

### Что должно работать

1. Зарегистрированный пользователь может подписаться на другого пользователя и отписаться
2. Можно посмотреть список подписчиков и подписок любого пользователя (пагинированно)
3. Домашняя лента показывает твиты от пользователей, на которых подписан, в обратном хронологическом порядке
4. Нельзя подписаться на себя
5. При подписке создаётся уведомление (если подписался не ты на себя)

---

## Критерии приёмки

- [ ] POST /users/{id}/follow → 204, подписка создана
- [ ] DELETE /users/{id}/follow → 204, подписка удалена
- [ ] POST /users/{id}/follow с id = себе → 400
- [ ] GET /users/{id}/followers → пагинированный список подписчиков
- [ ] GET /users/{id}/following → пагинированный список подписок
- [ ] После подписки твиты автора появляются в ленте
- [ ] GET /timeline/home → пагинированная лента, отсортированная по времени
