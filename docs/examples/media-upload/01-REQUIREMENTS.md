# Шаг 1: Бизнес-требования

> Пишет **человек** (PM, владелец продукта, тимлид).
> Может быть 3–10 строк. Никакой архитектуры, кода, деталей реализации.

---

## Требование

Пользователи могут загружать изображения в твиты.
Изображения хранятся в S3-совместимом хранилище (MinIO для dev, Cloudflare R2 / AWS S3 для prod).
В теле твита изображение отображается как URL.

### Что должно работать

1. При создании твита можно прикрепить до 4 изображений
2. Каждое изображение не больше 10 MB
3. Поддерживаемые форматы: JPEG, PNG, GIF, WebP
4. URL изображения возвращается в ответе на создание твита
5. Неавторизованные пользователи могут смотреть изображения

---

## Критерии приёмки

- [ ] POST /tweets с изображениями → 201 + твит с URLs
- [ ] GET /media/{id} → файл с правильным Content-Type
- [ ] Файл больше 10 MB → 413 Payload Too Large
- [ ] Неподдерживаемый формат → 400 Bad Request
- [ ] Больше 4 изображений → 400 Bad Request
