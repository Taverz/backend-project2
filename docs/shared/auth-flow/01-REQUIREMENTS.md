# Auth — бизнес-требования

---

## Требования

1. Новый пользователь может создать аккаунт (регистрация)
2. Зарегистрированный пользователь может войти (логин)
3. После входа пользователь получает access token (15 мин) + refresh token (7 дней)
4. Access token автоматически обновляется через refresh token при 401
5. Без токена пользователь видит только публичные страницы (логин, регистрация, твиты)
6. С токеном пользователь видит защищённые страницы (лента, профиль, уведомления)

---

## Критерии приёмки

- [ ] POST /auth/register → 201 + user + tokens
- [ ] POST /auth/login → 200 + user + tokens
- [ ] Неверный пароль → 401
- [ ] Дубликат email → 409
- [ ] Невалидный username → 400
- [ ] После логина /register → access token работает для 🔒 эндпоинтов
- [ ] При 401 → refresh → новый access token
- [ ] При неудачном refresh → logout → /login
- [ ] Все публичные страницы доступны без токена
- [ ] Все защищённые страницы → редирект на /login без токена
