# Step 0 — Project Brief

> Самый верхний слой. Без этого нельзя двигаться дальше.

## Что в этом шаге

| Файл | Что внутри | Когда читать |
|------|-----------|-------------|
| [`VISION.md`](VISION.md) | Связный рассказ о Bable: суть, цели, ожидаемые результаты. На простом языке. | **Первым делом** — если ты только зашёл в проект |
| [`PROJECT-BRIEF.md`](PROJECT-BRIEF.md) | Структурный brief: persona summary, MVP scope, open questions, Expertise Score механика | После VISION — для структурных деталей |
| [`PRINCIPLES.md`](PRINCIPLES.md) | 10 принципов: tone, a11y, privacy, moderation, performance, density, trust, mobile-first, i18n, AI | Перед каждым design решением — как фильтр |
| [`COPY-GUIDE.md`](COPY-GUIDE.md) | **Как Bable пишет**: buttons, errors, empty states, placeholders, labels. Конкретные шаблоны и banlist фраз | Когда пишешь любой текст в UI |

## Связь

```
VISION ─────► "что и зачем"
  │
  └─ детали ──► BRIEF ──► "как организовано"
                  │
                  └─ правила ──► PRINCIPLES ──► "как именно делаем"
                                    │
                                    └──► research / strategy / design
```
